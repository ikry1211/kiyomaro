## 3. 分析フロー

### 3.1. 全体フロー

```
Phase 1: データ収集（定量）
  ├─ Google Sheets から当月の財務データ取得
  ├─ Rakucoアクティビティ集計から営業KPI取得
  └─ 前年同月のデータ取得（前年比算出用）

Phase 2: データ収集（定性）
  ├─ Slack チャンネル分類スクリプト実行（活動状況の把握）
  ├─ 優先チャンネルの会話取得・要約
  └─ Notion からプロジェクト進捗・議事録取得

Phase 3: 分析・統合
  ├─ 財務サマリー作成（CF予測・月次PL）
  ├─ 事業別損益分析（受託・Rakuco・はなえみ）
  ├─ Rakuco営業KPI分析（新規・解約・ARR推移）
  ├─ Slackの定性情報と定量データの突合
  └─ 構造的リスク分析

Phase 4: レポート生成
  ├─ content/monthly/YYYY-MM.mdx として出力
  └─ ユーザーレビュー → Git PR → マージ
```

### 3.2. Phase 1: 定量データ収集

| # | 操作 | ツール | 取得範囲 |
|:---|:---|:---|:---|
| 1-1 | CF予測シート読み取り | `read_sheet` | `月次損益!A1:P50` |
| 1-2 | 事業別PL読み取り | `read_sheet` | 受託・Rakuco・はなえみ各タブ |
| 1-3 | Rakucoアクティビティ | `read_sheet` | 事業売上リスト |
| 1-4 | 前年データ（必要時） | `read_sheet` | 2024年度シートから同月 |

> **⚠ GEMINI.md ルール:** 取得時は広めの範囲（最低100行）を指定し、最終行の日付を必ず確認。未来月の0値は「未入力」として扱う。

### 3.3. Phase 2: 定性データ収集

| # | 操作 | ツール | 備考 |
|:---|:---|:---|:---|
| 2-1 | 優先チャンネルの会話取得 | Slack MCP `get_channel_history` | チャンネルマップの上位5-8ch |
| 2-2 | Notionプロジェクト検索 | Notion MCP `search` → `fetch` | Rakuco・はなえみ関連 |

> **チャンネル分類スクリプト** は分析とは別に **月初に1回実行** し、チャンネルマップ（`slack-channel-map.mdx`）を更新する。分析時にはマップを参照して優先チャンネルから会話を取得する。

### 3.4. Phase 3: 分析観点

| セクション | 分析内容 | データソース |
|:---|:---|:---|
| **財務サマリー** | CF推移・月次収支・資金繰り | Sheets（予算管理） |
| **事業別損益** | 受託・Rakuco・はなえみの損益と前月比 | Sheets（事業別タブ） |
| **営業KPI** | 新規獲得・解約・ARR・有料社数推移 | Sheets（アクティビティ）+ Slack（営業ch） |
| **CS/顧客動向** | 解約理由・クレーム傾向・サポート工数 | Slack（CS ch）+ Notion |
| **プロジェクト進捗** | 開発リリース・新機能・課題 | Slack（dev ch）+ Notion |
| **リスク分析** | 構造的リスクの更新と新規リスクの特定 | 全ソース統合 |
| **アクション** | 次月の優先アクションリスト | 分析結果から導出 |

## 4. レポート構成（テンプレート）

`content/monthly/YYYY-MM.mdx` の標準構成：

```
1. 財務サマリー
   1.1. キャッシュフロー予測
   1.2. 月次収支

2. 事業別損益
   2.1. 受託事業
   2.2. Rakuco事業
   2.3. 合計損益

3. Rakuco 営業KPI
   - 直近6ヶ月実績
   - 計画値との比較

4. 構造的リスク分析

5. 次月アクション

6. 総括
```

---

## 5. 運用ルール

| ルール | 内容 |
|:---|:---|
| **実施頻度** | 月初（当月データ確定後） |
| **データ鮮度** | 3ヶ月以上古いデータは「データ不足」として扱う |
| **推測の禁止** | データソースを明示し、推測でコメントしない |
| **未来月の扱い** | 0値は「未入力」と「実績0」を区別する |
| **レビューフロー** | AI生成 → ユーザーレビュー → Git PR → マージ |
| **Slackデータの扱い** | 個人名は匿名化、会話内容は要約して引用 |
