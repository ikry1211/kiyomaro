# 補助金・助成金戦略

> **ステータス:** 調査完了 → 方針決定待ち
> **最終更新:** 2026-03-30
> **判断待ち:** AI開発スタイルとの相性を踏まえた優先順位の確定

---

## 1. 前提: SOCTの開発スタイルと補助金の相性

### 1.1. 構造的な相性問題

SOCTの開発モデルは**「CEO + AI + 少数外注」の超低コスト・高速イテレーション型**であり、従来の補助金制度が想定する「外注に数百万〜数千万」の開発とは根本的に異なる。

```
SOCTの開発モデル（現状）:
━━━━━━━━━━━━━━━━━━━━━
  CEO（池田） × AIペアプロ → 設計・実装の大半
  外注（辻岡・高木等）     → Rakuco保守・特定機能
  
  特徴:
  ✅ 開発コスト: 極めて低い（AI活用で1人月≒数人月の生産性）
  ✅ イテレーション速度: 思いついたら即実装→即検証
  ✅ 方向転換の自由度: いつでも方針変更できる
  ✅ 認知負荷: 最小（自分で作るので仕様書不要）


補助金が求める開発モデル:
━━━━━━━━━━━━━━━━━━━━━
  事前に詳細な事業計画 → 審査 → 交付決定 → 着手 → 報告
  
  制約:
  ❌ 交付決定前の着手禁止 → 開発が数ヶ月止まる
  ❌ 事前の詳細計画が必要 → アジャイル開発と相反
  ❌ 計画変更に手続きが必要 → ピボットが遅くなる
  ❌ 効果報告義務（3〜5年） → 継続的な管理コスト
  ❌ 対象経費の証拠書類管理 → 事務負荷
```

### 1.2. 検証①: 並行でPoC進めながら申請できるか？

> **結論: できない。** ものづくり補助金・新事業進出補助金ともに、交付決定前の着手は補助対象外。並行開発は制度上不可。

```
補助金のタイムライン（ものづくり補助金の場合）:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  申請      採択      交付申請    交付決定    ここから着手OK
  5/8  →  6〜7月  →  手続き  →  7〜8月  →  ────────→
                                    ↑
                                    この前にやったことは
                                    全て補助対象外
```

| 質問 | 回答 |
|:--|:--|
| **申請しながらPoCを自費で進められる？** | 進めること自体は自由。ただし**その費用は補助対象外**。不採択なら全額自腹（当たり前）、採択されてもPoC分は補助されない |
| **事前着手承認制度はある？** | **ものづくり補助金にはない**（2026年現在）。旧・事業再構築補助金にはあったが、新事業進出補助金で引き継がれたかは公募要領で要確認 |
| **PoCと補助対象を分けられる？** | 理論的には可能だが、「PoCで検証済み → 本開発を補助金で」という計画は**審査で「すでに着手済み」と見なされるリスク**がある |

> **SOCTにとっての意味:** はなえみダイレクトを7月にローンチしたいのに、補助金を使おうとすると**8月まで正式着手できない**。しかも自費で先にPoC進めた部分は補助されない。時間も金も中途半端になる。

### 1.3. 検証②: 自社開発（CEOがAIで作る）は補助対象になるか？

> **結論: ならない。** 代表者の人件費は補助対象外。CEOが自分でコードを書く時間はゼロ円換算。

```
SOCTの開発コスト構造と補助対象:
━━━━━━━━━━━━━━━━━━━━━━━

  CEO（池田）の開発工数     → ❌ 補助対象外（代表者の人件費）
  従業員の開発工数           → ❌ 補助対象外（人件費は対象外 ※ものづくり補助金）
  AI API利用料（Claude等）   → △ クラウド利用料として認められる可能性あり（月¥1〜3万）
  インフラ費用（Vercel等）   → △ 同上（月¥1〜3万）
  外注費（辻岡・高木等）     → ✅ 補助対象
  機械装置・システム構築費    → ✅ ただし税込50万円以上の設備投資が必須条件
```

| ルール | 内容 |
|:--|:--|
| **代表者（社長）の人件費** | 補助対象外。合同会社の代表社員も同様 |
| **従業員の人件費** | ものづくり補助金では補助対象外 |
| **外注費のみの申請** | **不可**。ものづくり補助金は「税込50万円以上の設備投資（機械装置・システム構築費）」の計上が必須 |
| **AI APIコスト** | 「クラウドサービス利用料」として認められる可能性あるが、SOCTの場合月¥数万で金額小 |

> **SOCTにとっての意味:**
> - はなえみダイレクトのPhase 1は**池田さんが自分でAIと作る** → 補助対象の経費がほぼ存在しない
> - 外注費（辻岡さん等）は補助対象だが、Rakuco保守が中心で、はなえみダイレクトの外注は限定的
> - **仮に申請しても、補助対象として計上できる経費が少なすぎて割に合わない**

### 1.4. コスパの逆転現象

| 項目 | 補助金なし（現状） | 補助金あり |
|:--|:--|:--|
| **開発着手** | 即日 | 申請〜交付決定まで**2〜4ヶ月待ち** |
| **方針転換** | 自由 | 計画変更手続きが必要 |
| **事務コスト** | ¥0 | 申請書作成¥50〜100万（外注の場合）+ 報告義務 |
| **CEO認知負荷** | 低 | **高**（申請・報告が頭にある状態が数年続く） |
| **開発費** | 月¥5〜10万（AI API + インフラ） | — |
| **CEOの開発工数** | 補助対象にならない | 補助対象にならない（同じ） |

> **核心:** SOCTの開発費がそもそも低く、かつCEOの自社開発は補助対象外。**補助金で得られる金額 vs 失う時間・自由度・認知負荷**のトレードオフが成立しない。

### 1.5. 補助金が「合う」ケースと「合わない」ケース

| ケース | 相性 | 理由 |
|:--|:--|:--|
| ❌ CEO自身がAIで開発するもの | **合わない** | コストが低すぎ＋人件費対象外。速度を殺す |
| ❌ 仮説検証フェーズの新規事業 | **合わない** | ピボットの自由度が最重要。事前着手不可 |
| ⚠️ 外注開発費が大きいもの | **場合による** | 外注費¥500万超なら検討余地あり。ただし50万以上の設備投資も必要 |
| ✅ 販促・マーケティング費用 | **合う** | コーポレートサイト制作、展示会出展など |
| ✅ 顧客の導入支援（ベンダー登録） | **非常に合う** | SOCT側のコスト≒¥0。営業武器になる |
| ✅ 人件費系の助成金 | **合う** | 正社員化・テレワーク導入など |
| ✅ M&A実行時 | **合う** | FA費用・DD費用など大型支出時 |

## 2. SOCTに本当にフィットする制度

定番の補助金（ものづくり、IT導入）は既知なので割愛。SOCTの特殊なプロファイルに刺さる制度を中心に整理する。

### 2.1. ⭐⭐⭐ 研究開発税制（試験研究費の税額控除）— 詳細検証

> **補助金ではなく税額控除。申請不要・交付決定待ち不要・開発速度への影響ゼロ。SOCTのAI開発スタイルと最も相性が良い。**

#### 制度の全体像（2つのルートがある）

| 制度 | 控除率 | 控除上限 | 適用要件 | SOCTとのフィット |
|:--|:--|:--|:--|:--|
| **中小企業技術基盤強化税制** | **12〜17%** | 法人税額の25% | 青色申告 + 中小企業であること | ⭐⭐⭐ 現実的 |
| **戦略技術領域型（AI特例）** | **40〜50%** | 法人税額の10%（別枠） | 「重点研究開発計画」の主務大臣認定 | ⚠️ ハードル高い |

##### 「戦略技術領域型」の現実的なハードル

```
控除率40〜50%は魅力的だが:
  ❌ 「重点研究開発計画」を主務大臣（経産大臣等）に認定してもらう必要がある
  ❌ 産業技術力強化法に基づく正式な計画認定 → 大企業・ディープテック向け
  ❌ SOCTのようなSaaSスタートアップが単独で認定を受けるのは実務上困難
  
  → SOCTは「中小企業技術基盤強化税制（12〜17%）」を使うのが現実解
  → 戦略技術領域型は「知っておく」レベル。将来規模が大きくなった場合に再検討
```

#### SOCTの支出で「試験研究費」に該当するもの

| 費目 | 対象可否 | 根拠 | SOCTでの具体例 |
|:--|:--|:--|:--|
| AI API利用料 | ✅ | 「新技術の開発に必要なクラウドサービス利用料」 | Claude API, OpenAI API（月¥1〜5万） |
| クラウドインフラ | ✅ | 同上 | Vercel, Supabase, AWS（月¥1〜3万） |
| 外注の開発費 | ✅ | 「委託研究費」として明確に対象 | 辻岡さん等への新機能開発委託 |
| プロトタイピング用の有料サービス | ✅ | 試作に必要な費用 | テスト用ツール・データ等 |
| 代表者（池田さん）の人件費 | **❌〜⚠️** | 合同会社の代表社員 → 対象外の可能性が高い | — |
| 従業員の人件費 | ✅ | 試験研究に専従する時間分 | 将来の正社員エンジニアの研究開発工数分 |
| 保守・運用・バグ修正 | ❌ | 「既存製品の維持」は試験研究ではない | Rakucoの日常保守 |
| 顧客カスタマイズ | ❌ | 同上 | — |

##### 国税庁が求める「試験研究」の要件（サービス開発の場合）

```
「試験研究費」として認められるために、以下のプロセスを踏む必要がある:

  ① データの準備・取得
  ② データの分析（AI等の活用）
  ③ サービスの設計
  ④ サービスの開発
  ⑤ サービスの適用・再現性の確認

→ SOCTの開発プロセス（はなえみダイレクトのAIマッチング等）は
  このフレームワークにかなり近い ✅
```

#### SOCTの年間試験研究費シミュレーション

```
仮定: SOCTの年間開発関連支出
━━━━━━━━━━━━━━━━━━━━━━━

  AI API利用料:        ¥360,000/年（月¥30,000）
  クラウドインフラ:     ¥240,000/年（月¥20,000）
  外注開発費（R&D分）:  ¥1,200,000/年（新機能開発の外注分）
  ────────────────────────────
  試験研究費 合計:      ¥1,800,000/年

税額控除（中小企業技術基盤強化税制: 12%の場合）:
  ¥1,800,000 × 12% = ¥216,000

税額控除（試験研究費増加で17%適用の場合）:
  ¥1,800,000 × 17% = ¥306,000
```

> **正直な評価:** 現状の開発費規模（月¥5〜10万）だと、控除額は年¥20〜30万程度。**額自体は大きくない**が:
> - **申請の手間がほぼゼロ**（確定申告に別表を追加するだけ）
> - **開発スピードに一切影響しない**
> - **外注費が増えた年はそのまま控除額も増える**（スケーラブル）
> - **赤字期に計上した分を黒転後に繰り越せる**（2026年改正）

#### 赤字→黒転期に活きる「繰越控除」の仕組み

```
2026年度改正で新設:
  「当期の法人税額で控除しきれない分 → 3年間繰越可能」

  SOCTのシナリオ:
  ─────────────
  FY2026: 赤字（法人税ゼロ）→ 試験研究費¥180万を計上 → 繰越
  FY2027: 赤字（法人税ゼロ）→ 試験研究費¥240万を計上 → 繰越
  FY2028: 黒字転換・法人税¥200万発生
          → FY2026〜2028の累計試験研究費で税額控除
          → 法人税¥200万 × 25%（控除上限） = ¥50万が控除可能

  ⚠️ ただし適用条件あり:
  「当期の試験研究費 > 前期の試験研究費」であること（研究開発投資を増やしている必要）
```

> **判断:** 額は小さいが、**手間ゼロ・制約ゼロ**で開発費の一部を取り戻せる。将来外注費が増えた時に真価を発揮する。**今からやるべきは「記録の習慣づけ」**。

#### 今すぐやるべきこと

| # | アクション | 工数 |
|:--|:--|:--|
| 1 | 顧問税理士に「AI API費用・外注費を試験研究費で計上できるか」を確認 | 30分の電話 |
| 2 | 開発ログの記録を開始（何を開発したか、目的は何か、どのAPIを使ったか） | 日次5分 |
| 3 | AI API・クラウド費用の請求書を「研究開発」と「運用」に区分して保存 | 月次10分 |

> 税理士への相談ポイント: **「SaaS開発のAI API利用料は試験研究費に該当するか」「合同会社の代表社員の開発工数は対象になるか」**の2点を確認。

#### R&D判断基準（SOCTの開発業務における仕分けルール）

外注費・従業員の工数を「R&D（試験研究費）」と「保守・運用」に仕分ける際の判断基準:

##### ✅ R&D（試験研究費）に該当するもの

| カテゴリ | 具体例 | 判断理由 |
|:--|:--|:--|
| **新機能の企画・設計・実装** | RakucoにAI記録アシスト機能を追加 | 技術的に新しい課題の解決 |
| **新プロダクトの開発** | はなえみダイレクトのマッチングエンジン構築 | 新サービスの創出 |
| **AIモデル・アルゴリズムの構築** | LLMを使った福祉記録の自動生成ロジック | 新技術の研究開発 |
| **アーキテクチャの大幅刷新** | マルチテナント化、DBスキーマ再設計 | 技術基盤の革新 |
| **新しい計算エンジンの開発** | 報酬改定に伴う新しい計算ロジックの設計・実装 | 新しい技術的課題の解決 |
| **PoC・プロトタイピング** | 新しいUI/UXパターンの技術検証 | 試作・実験 |
| **データ分析基盤の構築** | 顧客分析ダッシュボードの設計・開発 | 新サービス開発のための分析 |

##### ❌ R&Dに該当しないもの（保守・運用）

| カテゴリ | 具体例 | 判断理由 |
|:--|:--|:--|
| **バグ修正** | 既存機能の不具合対応 | 既存製品の維持 |
| **UI微調整** | ボタン配置の変更、色の調整 | 技術的に新しくない |
| **顧客対応の個別カスタマイズ** | 特定事業所向けの設定変更 | 汎用的な研究開発ではない |
| **定期メンテナンス** | サーバー更新、ライブラリアップデート | 運用業務 |
| **既存帳票の修正** | レイアウト微修正 | 既存機能の維持 |
| **データ移行・初期設定** | 新規顧客のセットアップ | CS業務 |

##### ⚠️ グレーゾーン（税理士と要確認）

| カテゴリ | 具体例 | ポイント |
|:--|:--|:--|
| **制度改定への対応** | 報酬改定の反映 | 単純な数値変更→保守 / 新しい計算エンジン開発→R&D |
| **パフォーマンス改善** | 大規模なリファクタリング | 技術的に新しいアプローチを使うならR&D寄り |
| **既存機能の大幅改修** | 画面の全面リニューアル | 技術的挑戦があればR&D、見た目だけなら保守 |

##### 運用ルール

```
月次の仕分け手順:
━━━━━━━━━━━━━━━

  1. 外注先（高木、上野、玉利等）の月次稼働を確認
  2. 各人の作業内容を「R&D」「保守」「その他」に分類
  3. R&D比率を算出（例: 高木さん 月40h中 R&D 20h = 50%）
  4. 外注費 × R&D比率 = 試験研究費として計上

  記録すべき情報:
  ├── 誰が（外注先名）
  ├── 何を開発したか（機能名・プロジェクト名）
  ├── なぜそれが「試験研究」か（技術的に新しい点）
  └── 何時間使ったか（R&D工数）

  保存場所: スプレッドシート or Notionに月次記録
  頻度: 月1回（外注費の請求確認時にまとめて記録）
  工数: 月15〜30分
```

### 2.2. ⭐⭐⭐ 在宅就業障害者特例調整金・特例報奨金

> **SOCTがB型事業所に業務委託している構造そのものが、顧客企業の制度活用に直結する。**

| 項目 | 内容 |
|:--|:--|
| **制度名** | 在宅就業障害者特例調整金 / 特例報奨金 |
| **管轄** | JEED（高齢・障害・求職者雇用支援機構） |
| **対象** | 障害者（個人）または「在宅就業支援団体」に業務発注した企業 |
| **支給額** | 調整金: ¥21,000 × (発注額÷35万円) / 報奨金: ¥17,000 × 同 |

#### SOCTの事業との関連

```
SOCTのフォーム営業フロー（現状）:
  企業クライアント → SOCT → B型事業所（障害福祉）→ フォーム送信

このスキームにおいて:
  ❌ B型利用者はSOCTの「雇用」ではない → 法定雇用率にカウント不可
  ❌ SOCTがこの制度を直接使えるわけではない

しかし:
  ✅ SOCTの「顧客企業」がB型事業所に業務を発注する場合、
     顧客側が特例調整金の対象になりうる
  ✅ はなえみダイレクトの法人顧客（障害者を雇用する側）にとって
     「B型事業所への発注 → 特例調整金で実質コスト軽減」という
     営業トークが成立する可能性
```

#### 2026年7月の法定雇用率引き上げ（2.5% → 2.7%）との連動

```
法定雇用率2.7% → 37.5人以上の企業が義務対象に拡大
  → 多くの中小企業が障害者雇用の対応に追われる
  → 「はなえみダイレクト」のマッチングPFの需要が高まる
  → 顧客企業に「在宅就業障害者特例調整金」の活用をセットで提案
  → はなえみダイレクトの付加価値向上
```

> **アクション:** この制度の詳細をはなえみダイレクトの営業資料に組み込めるか調査。顧客企業のメリットとして訴求。

### 2.3. ⭐⭐ 東京都 新製品・新技術開発助成事業 — 詳細検証

> **国の補助金と違い「直接人件費」が対象。ソフトウェア試作品の開発も明確に対象。ただしスケジュール拘束は残る。**

#### 制度の詳細

| 項目 | 内容 |
|:--|:--|
| **実施** | 公益財団法人 東京都中小企業振興公社 |
| **助成額** | **最大2,500万円** |
| **助成率** | 1/2（賃上げ計画策定で**3/4〜4/5**に引き上げ可能） |
| **対象** | 都内中小企業の新製品・新技術（ソフトウェア含む）の研究開発 |
| **申請受付** | **2026年3月27日〜4月17日（第1回）** ← 今まさに受付中 |
| **助成対象期間** | 2026年9月1日 〜 2028年5月31日（最長1年9ヶ月） |
| **交付決定** | 9月初旬予定 |

#### ものづくり補助金との比較（詳細版）

| 比較項目 | ものづくり補助金（国） | 東京都 新製品開発助成 |
|:--|:--|:--|
| 助成額 | 750〜1,250万円 | **最大2,500万円** |
| **直接人件費** | **❌ 対象外** | **✅ 対象（上限あり）** |
| 委託・外注費 | ✅ 対象 | ✅ 対象 |
| 設備投資必須 | ✅ 50万以上必須 | 不要（研究開発費中心） |
| 設備費 | ✅ | ✅ 機械装置・工具器具費 |
| 産業財産権（特許） | ✅ | ✅ 出願・導入費 |
| 専門家指導費 | — | ✅ 対象 |
| 助成率 | 1/2〜2/3 | 1/2（賃上げで**3/4〜4/5**） |
| 交付決定前着手 | **❌ 不可** | **❌ 不可** |
| 効果報告義務 | 3〜5年 | あり（ただし国ほど重くない） |
| 審査方法 | 書面＋口頭 | 書面＋面接 |

> **最大の違い:** **「直接人件費」が対象**。従業員の研究開発に従事した時間分の人件費を助成対象にできる。ものづくり補助金にはないメリット。

#### SOCTへのフィット判定

```
✅ フィットする点:
  ├── ソフトウェアの試作品設計・製作・試験が明確に対象
  ├── 人件費が対象 → 将来エンジニアを雇用した場合に効果大
  ├── 外注費も対象 → 辻岡さん等への開発委託
  ├── 助成率が賃上げ計画で最大4/5（80%）に
  └── 期間が最長1年9ヶ月 → 長めの開発を支援

❌ フィットしない点:
  ├── 交付決定が9月 → 9月1日以降の契約・支出のみ対象
  │   → はなえみダイレクトPhase 1（4〜7月開発）は対象外
  ├── 「達成目標」の設定が必須 → 未達成なら助成金不交付
  ├── CEOの人件費は対象外（代表者は「直接人件費」に含まれない）
  └── 都内に本店登記が必要（SOCTが該当するか要確認）
```

#### 現実的な活用シナリオ

```
シナリオ: はなえみダイレクト Phase 2 / Rakuco大型改修 に適用
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

タイムライン:
  4月中旬: 申請（今回の受付に間に合わせる場合）
  6〜8月:  審査（面接あり）
  9月:     交付決定 → ここから経費計上可能
  9月〜:   Phase 2の開発着手（外注比率高めのフェーズ）

想定経費:
  外注開発費:      ¥3,000,000（新機能の大型開発）
  人件費（従業員）: ¥2,000,000（9月〜翌5月の9ヶ月分）
  クラウド費用:     ¥500,000
  ──────────────────────────
  合計:            ¥5,500,000
  
  助成額（1/2の場合）:   ¥2,750,000
  助成額（4/5の場合）:   ¥4,400,000 ← 賃上げ計画ありの場合
```

#### 判断: 今回（4/17締切）に申請すべきか？

```
申請する場合のメリット:
  ✅ 9月以降の開発費を最大80%カバーできる可能性
  ✅ 人件費が対象 → 将来の採用計画と連動
  ✅ ものづくり補助金より柔軟

申請する場合のリスク・コスト:
  ❌ 申請書の準備（池田さんの工数: 20〜40時間）
  ❌ 面接対応の工数
  ❌ 9月まで対象経費の発生を待つ必要あり → Phase 1には使えない
  ❌ 不採択のリスク（採択率は30〜40%程度）
  ❌ 達成目標の拘束 → ピボットしづらい
```

> **結論:** 4月17日の申請は**Phase 1の開発速度を犠牲にしてまで準備する価値はない**。ただし、Phase 2で外注費が増えるフェーズ（9月以降）には有効。**第2回の公募（秋頃）を狙う方が現実的**。その頃にはPhase 1の実績もあり、事業計画の説得力が増す。

#### SOCTが都内に本店登記しているか — 要確認

本制度は「都内に本店または主たる事業所を有する中小企業」が対象。SOCTの登記が都内にあるかを確認するのが先決。

### 2.4. ⭐⭐ はなえみダイレクトの「顧客側」が使える助成金

> **SOCTが申請するのではなく、顧客企業が使える障害者雇用助成金の知識を営業武器にする。**

2026年7月の法改正で多くの企業が障害者雇用対応に追われる。はなえみダイレクトの顧客が使える助成金を把握しておくと営業に活きる。

| 助成金 | 対象 | 金額 | はなえみダイレクトとの連携 |
|:--|:--|:--|:--|
| **特定求職者雇用開発助成金** | 障害者をハローワーク経由で雇用 | 最大240万円/人 | マッチング後の雇用に適用 |
| **トライアル雇用助成金** | 試行的に雇い入れ | 月4万円×最大3ヶ月 | 「まず試してみたい」企業への訴求 |
| **障害者介助等助成金** | 職場環境整備 | 実費ベース | 職場定着支援のNakaraと連携 |
| **キャリアアップ助成金（障害者正社員化）** | 有期→正社員転換 | 最大120万円/人 | 長期定着のインセンティブ |

> **核心:** はなえみダイレクトが「マッチングして終わり」ではなく、**「この助成金が使えるので実質コスト削減できます」**とセットで提案できると、成約率と顧客満足度が上がる。

> **補助金ではなく融資だが、社会課題解決型の事業に低金利で融資。返済義務あるが自由度が高い。**

はなえみダイレクト（障害者雇用マッチング）は「社会課題解決型」に該当しやすい。補助金のような開発制約がなく、資金の使途も比較的自由。

## 3. 結論: SOCTの補助金・制度活用方針

```
軸1: 税制で開発費を取り戻す（← NEW: 最優先）
──────────────────────────────────────
└── 研究開発税制 → AI API費用・外注開発費の12〜17%を法人税控除

軸2: 顧客の導入ハードルを下げる（営業武器）
──────────────────────────────────────
├── IT導入支援事業者登録 → Rakuco / eラーニング
├── 障害福祉ICT補助金 → Rakuco顧客
├── 在宅就業障害者特例調整金 → はなえみダイレクト顧客
└── 障害者雇用助成金の知識 → はなえみダイレクト営業

軸3: 販促・人事に限定して補助金を使う
──────────────────────────────────────
├── 持続化補助金 → コーポレートサイト / 販促
├── キャリアアップ助成金 → 正社員化
└── M&A補助金 → 実行時

❌ やらないこと:
├── ものづくり補助金 → 開発速度を犠牲にする価値がない
└── 新事業進出補助金 → 同上
```

### 3.1. ⭐ IT導入支援事業者（ベンダー）登録 — 最優先

| **SOCTのコスト** | ¥0（登録は無料。事務対応の工数のみ） |

#### 営業インパクト

```
顧客（障害福祉事業所）にとって:
  ✅ Rakuco月額¥18,000 × 最大24ヶ月 = ¥432,000が補助対象
  ✅ はなえみラーニング利用料も補助対象
  ✅ 「補助金が使える」は導入の強い動機になる

SOCTにとって:
  ✅ 営業トーク: 「補助金で実質無料で導入できます」
  ✅ 成約率の向上（導入ハードルの低下）
  ✅ 競合knowbeが登録済みなら差別化消失を防止
  ✅ 公的認定によるブランド信頼性の向上
```

#### 注意事項
- **伴走支援義務**: 顧客の申請サポート、導入後の効果報告支援（年3回程度）
- **事務工数**: 石丸さん or CSチームで対応可能だが、初期の手続き学習コストあり
- **GビズIDプライムが必要**（SOCT側で取得済みか要確認）

> ⚠️ 競合knowbeがIT導入支援事業者に登録済みの場合、Rakucoが未登録だと**補助金の有無で比較負け**するリスクがある。要調査。
