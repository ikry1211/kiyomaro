## 1. 背景 — なぜ今ブランド戦略が必要か

株式会社SOCTは2017年の創業以来、受託開発を主軸としてきた。しかし2025年のRakuco事業譲受を起点に、自社プロダクト群（Rakuco・はなえみラーニング・はなえみダイレクト・Tetra LMS）を擁する企業へと構造転換が進んでいる。

| 変化 | 以前 | 現在 |
|:---|:---|:---|
| 会社形態 | 合同会社 SOCT（SOCT LLC.） | **株式会社 SOCT（SOCT Inc.）** |
| 事業構成 | 受託開発100% | 受託（開発・広告）+ SaaS + PF + マッチング |
| ブランド数 | SOCTのみ | SOCT / Rakuco / はなえみ○○ / Tetra LMS |
| 組織設計 | 開発チーム | 事業部制（Rakuco事業部 / 新規事業） |

**ブランド戦略が必要な理由:**
1. **複数ブランドの並行展開** — Rakucoに加え、はなえみダイレクト・はなえみラーニングが立ち上がる。一貫性と分離を先に定義する
2. **事業リスクの分散** — 1つのブランドに全事業を乗せず、独立して生存可能なブランド体系にする
3. **コーポレートサイトの乖離** — 現サイトは受託会社としての情報のみ。実態を反映した再設計の指針が必要

## 2. ブランドの二層構造

SOCTのブランド戦略は**コーポレートブランド**と**サービスブランド**の二層で設計する。

```
┌──────────────────────────────────────────────────────┐
│  コーポレートブランド: SOCT                            │
│  「想うより早く、アイディアを形に。」                     │
│  → 業種を問わず、テクノロジーで事業を形にする会社         │
│  → 受託（開発・広告）もここに属する                      │
├───────┬───────┬──────────────┬────────────────────────┤
│ Rakuco│はなえみ│はなえみ      │ Tetra LMS             │
│       │ﾀﾞｲﾚｸﾄ │ﾗｰﾆﾝｸﾞ       │                       │
│ 障害福│障害者雇│障害福祉職員  │ 研修PF                │
│ 祉SaaS│用ﾏｯﾁﾝｸﾞ│eラーニング   │ 基盤                  │
└───────┴───────┴──────────────┴────────────────────────┘
  サービスブランド: 各市場に特化したメッセージを持つ
```

| 層 | 対象 | 役割 |
|:---|:---|:---|
| **コーポレート（SOCT）** | 受託クライアント・投資家・パートナー | SOCTの能力と信頼性を伝える。業種を限定しない |
| **サービスブランド群** | 各サービスのエンドユーザー | 特定市場に最適化されたメッセージで訴求する |

> **なぜ二層か:** SOCTは障害福祉向けプロダクトと、業種を問わない受託事業を併せ持つ。コーポレートブランドを特定領域に閉じると、受託やTetra LMSの訴求が弱くなる。逆にサービスブランドを汎用にすると、各市場での訴求力が薄まる。二層にすることで両方を最適化できる。

## 3. コーポレートブランド — SOCT

### 3.1. ポジショニング

SOCTは**テクノロジーで事業を素早く形にする会社**。受託開発でクライアントのアイディアを形にし、自社プロダクトで自らのアイディアを形にしてきた。

| SOCTの能力 | 受託での発揮 | 自社プロダクトでの発揮 |
|:---|:---|:---|
| 企画→実装まで一気通貫 | InnoJin・住友商事等の新規事業開発 | Rakuco・はなえみの0→1立ち上げ |
| AI + PF設計 | — | Tetra LMS・はなえみダイレクトの設計 |
| グロース支援（広告運用） | 綿半・シェアダイン・ゴルフパフォーマンス | — |

### 3.2. コーポレートタグライン

> **「想うより早く、アイディアを形に。」**

クライアントのアイディアも、自分たちのアイディアも、速く形にする — これがSOCTの本質。業種を問わず適用できるコーポレートメッセージとして**継続**する。

### 3.3. コアバリュー

| バリュー | 意味 |
|:---|:---|
| **現場起点** | 机上の理論ではなく、現場の「不」から出発する。ヒアリングベースで作る |
| **テクノロジー × 人** | テクノロジーで効率化しつつ、人的支援で温度感を保つ |
| **持続可能な仕組み** | スケールしても品質が落ちない構造を優先する |
| **透明性** | 料金・データ・意思決定の根拠を可視化する |

### 3.4. コーポレートのトーン

| 要素 | 方向性 | 避けること |
|:---|:---|:---|
| **プロフェッショナル** | 技術力と実行力が伝わる表現 | 曖昧な主張・根拠のない約束 |
| **シンプル** | 短い文章、明快な構造 | 冗長な説明・過度な装飾 |
| **誠実** | できることとできないことを正直に | 煽り文句・過大な約束 |

## 4. サービスブランド群

### 4.1. 共通ビジョン（障害福祉プロダクト群）

> **「テクノロジーが届いていない現場を、仕組みで変える」**

Rakuco・はなえみダイレクト・はなえみラーニングの**3サービス**が共有するビジョン。コーポレートのミッションではなく、**障害福祉領域に向けたサービスブランド群の旗印**として位置づける。

> ⚠️ **Tetra LMSはこのビジョンの対象外。** Tetra LMSは業種を問わない汎用の研修PF基盤であり、コーポレート層に近い位置づけ。はなえみラーニングはTetra LMS基盤上で障害福祉向けに展開するサービスという関係。

### 4.2. ターゲットと提供価値

| ターゲット | 関わるサービス | 課題 | SOCTの回答 |
|:---|:---|:---|:---|
| **障害福祉事業所** | Rakuco / はなえみラーニング / はなえみダイレクト | 煩雑な事務・研修負荷・就職支援 | 業務SaaS + eラーニング + マッチングPF |
| **利用者（障害当事者）** | はなえみダイレクト | 就職活動の選択肢が限られる | PFでのマッチング。事業所と企業をつなぐ |
| **雇用義務のある企業** | はなえみダイレクト | 採用チャネルの確保 | PFでマッチング。定着支援付き |
| **研修事業者** | Tetra LMS | 自前で研修PFを構築できない | Shopify型基盤を提供 |

> **最重要ターゲットは障害福祉事業所。** 104社の実績を持つRakucoの既存顧客基盤が、はなえみラーニング・はなえみダイレクトのクロスセル先でもある。

### 4.3. サービス別メッセージング

| サービス | タグライン案 | 訴求ポイント |
|:---|:---|:---|
| **Rakuco** | 福祉の事務を、ラクにする。 | 報酬計算・記録管理の効率化。改定対応の安心 |
| **はなえみダイレクト** | つなぐ。はたらく。ささえる。 | 採用からフォローアップまで一気通貫 |
| **はなえみラーニング** | 学びで、現場を強くする | 法定研修のオンライン化。いつでもどこでも |
| **Tetra LMS** | 教える力が、価値になる。 | 知識の収益化。Shopify型基盤で自分のブランドで販売 |

### 4.4. サービスブランドのトーン

| 要素 | 方向性 | 避けること |
|:---|:---|:---|
| **温かさ** | 福祉の文脈に馴染む親しみやすさ | 冷たい企業語調・上から目線 |
| **専門性** | 福祉制度への深い理解が伝わる表現 | 業界用語の乱用 |
| **シンプルさ** | 事業所の担当者に伝わる平易な言葉 | 冗長な説明・テック用語の多用 |

### 4.5. 競合ポジショニング（障害福祉領域）

```
                 テクノロジー主導
                      │
                      │        ★ SOCT サービス群
                      │        （SaaS + AI + PF）
                      │
   汎用 ──────────────┼────────────── 障害福祉特化
                      │
           knowbe     │    北澤経由の人的支援
           かべなしクラウド│    株式会社はなえみ
                      │
                 人的サービス主導
```

| SOCTが戦わない領域 | 理由 |
|:---|:---|
| 放デイ・居宅介護等の汎用福祉 | knowbe等が先行。就労系特化で深掘りする |
| 人的コンサル・対面研修 | 株式会社はなえみが担う。SOCTはテック |
| 大企業向け人材紹介 | atGP・パーソル等が担う。SOCTはPF型 |

## 5. ブランド体系

### 5.1. ポートフォリオ構造

```
株式会社 SOCT（コーポレート）
│
├── 受託事業 ───── システム開発支援 + 広告運用（収穫事業）
│
├── Rakuco ───────── 障害福祉 業務管理SaaS（主力事業）
│
├── はなえみブランド ── SOCTの中長期ブランド
│   ├── はなえみダイレクト ── 障害者雇用マッチングPF
│   └── はなえみラーニング ── 障害福祉職員向けeラーニング
│
└── Tetra LMS ──────── 研修プラットフォーム基盤
```

### 5.2. ブランドの独立性（リスク分散設計）

| ブランド | 性格 | 独立性 |
|:---|:---|:---|
| **SOCT** | コーポレート・技術基盤 | — |
| **Rakuco** | 実務的・信頼性 | 単独で事業運営可能 |
| **はなえみ○○** | 温かみ・つながり | 単独で事業運営可能 |
| **Tetra LMS** | プロフェッショナル・基盤 | 単独で事業運営可能 |

> **設計意図:** 各ブランドが独立して事業運営できる構造。1つのブランド・事業に問題が発生しても、他のブランドが影響を受けずに存続できるリスク分散設計。

### 5.3. 補完関係 — SOCTと株式会社はなえみ

| | SOCTの「はなえみ」サービス群 | 株式会社はなえみ |
|:---|:---|:---|
| 担う範囲 | テック（SaaS、プラットフォーム） | 人的サービス（顧問、フォローアップ） |
| 収益帰属 | SOCT (100%) | 株式会社はなえみ（池田49%） |
| ブランド使用 | SOCTが商標保有。使用許諾で共有 | 社名として使用 |
| シナジー | PF経由のマッチング → コンサル案件化 | 増分コンサルの50%をSOCTに分配 |

> 詳細な商標管理・使用許諾については [知的財産戦略](./ip-strategy) を参照。

> 具体的なカラーパレット・書体はコーポレートサイトリニューアル時に確定する。

| 要素 | 方向性 |
|:---|:---|
| **コーポレートカラー** | 信頼感のあるブルー系（現状踏襲しつつ刷新検討） |
| **はなえみカラー** | 温かみ・やさしさを表現する暖色系 |
