# PR・情報発信戦略

> **ステータス:** 方針確定 → 実行準備
> **最終更新:** 2026-04-27
> **関連:** [テックブログ戦略](./tech-blog-strategy) — 将来的な統合候補 | [ブランド戦略](./brand-strategy) — コーポレートブランドの発信手段

---

## 1. 背景と課題

### 1.1. 現状

SOCTの案件獲得経路は以下の3つに限られている。

| 経路 | 事例 | 課題 |
|:---|:---|:---|
| **CEO紹介・既存人脈** | 中川様、Rigna、シェアダイン | 池田の人脈に依存。能動的に増やせない |
| **過去商談の掘り起こし** | 中川様（1年前の商談から再連絡） | 偶発的。仕組み化されていない |
| **テレアポ** | コンサル事業 | Rakuco向け。受託・広告向けではない |

**能動的に見込み客と接点を維持し、SOCTを想起してもらう手段がない。**

### 1.2. テックブログからの方針転換

当初はテックブログ（blog.soct.jp）を検討したが、以下の理由で現段階では見送り。

| 課題 | 詳細 |
|:---|:---|
| **体制を公開できない** | 「1人+AI」の実態を出すと、クライアントへの提案書と矛盾する |
| **定量的な成果データがない** | 「やってみました」記事にしかならず、読者を動かせない |
| **主力プロダクトが未リリース** | Tetra LMSのリリース前に書いても「画餅」に見える |

→ 深い技術記事を書く代わりに、**小さなニュースの継続発信 + ニュースレター配信**の方がSOCTの現状に適合する。

---

## 2. 戦略の全体像

```
小さなネタを作る（随時）
  ├── note記事
  ├── X（旧Twitter）投稿
  ├── プレスリリース（PR TIMES等）
  └── コーポレートサイト更新
      ↓
  ニュースレターに集約して配信（四半期に1回）
      ↓
  既存クライアント + 過去商談先
```

### 2.1. テックブログとの違い

| | テックブログ（見送り） | **PR + ニュースレター（本施策）** |
|:---|:---|:---|
| コンテンツ | 技術的な深い記事 | 小さなニュースの積み重ね |
| 制約 | 体制を出せない・成果データがない | 事実ベースのお知らせなので制約が少ない |
| 配信先 | 不特定多数（SEO頼り） | 既存の関係先に直接届く |
| 頻度 | 月2本（ハードル高い） | ネタができた時に随時 + 四半期まとめ |
| 更新が止まるリスク | 🔴 高い（放置→逆効果） | 🟢 低い（四半期に1通で十分） |

## 3. チャネル設計

### 3.1. 発信チャネル（アウトプット先）

| チャネル | 用途 | 頻度 |
|:---|:---|:---|
| **コーポレートサイト** | 実績ページ・お知らせ | 更新があった時 |
| **note** | まとまった記事（導入事例・取り組み紹介） | 月0〜1本 |
| **X** | 短文の所感・ニュース共有・note記事の告知 | 週1〜2投稿 |
| **プレスリリース** | 大きなニュース（リリース・提携・100社突破等） | 不定期 |
| **ニュースレター** | 上記すべてを集約して配信 | **四半期に1回** |

### 3.2. ニュースレター

#### 配信対象

| セグメント | 対象 | 想定人数 |
|:---|:---|:---|
| **既存クライアント** | Rakuco顧客の法人担当者、受託・広告のクライアント | 〜30名 |
| **過去商談先** | 商談したが成約に至らなかった企業 | 〜20名 |
| **パートナー・紹介者** | CEO紹介元、業務提携先 | 〜10名 |

#### 内容のフォーマット

```
SOCT Newsletter — 2026年 Q2

■ [ニュース] Rakuco導入事業所が100社を突破しました
■ [ニュース] 株式会社に組織変更しました
■ [開発] 新サービス「Tetra LMS」を開発中です
■ [更新] コーポレートサイトにWorksページを追加しました

━━━━━━━━━━━━━━━━━━━━━━
株式会社SOCT
https://soct.co.jp
```

> 1通に3〜5トピック。各トピック2〜3行 + リンク。簡潔に。

#### ツール

**自社開発が決定。** はなえみダイレクトのMailchimp（30,000件配信）を自前ツールに置き換え、Rakuco・SOCT・コンサル事業でも共用する。

| 項目 | 内容 |
|:---|:---|
| **配信基盤** | Resend or Amazon SES |
| **月額コスト** | ¥3,000〜（Mailchimp比で年¥38万以上削減） |
| **対象** | はなえみ30,000件 + Rakuco 104社 + SOCT ~50名 |
| **詳細** | [バックログ #22](/backlog) |

## 4. コンテンツ設計

### 4.1. ネタの分類

| 種別 | 内容 | 例 | 発信チャネル |
|:---|:---|:---|:---|
| **📢 ニュース** | 会社の出来事 | 株式会社化、100社突破、新サービス発表 | プレスリリース + note + X |
| **🏗️ 進捗報告** | 開発中のプロダクト | Tetra LMS開発中、新機能追加 | X + ニュースレター |
| **💡 所感・考察** | 業界・AIに関する見解 | 福祉DXの動向、AI活用の所感 | X |
| **📋 実績更新** | サイトの更新 | Worksページ追加、事例記事公開 | X + ニュースレター |

### 4.2. 発信時の制約（必ず守ること）

| ❌ 出さない | 理由 |
|:---|:---|
| 社員数・体制の詳細 | クライアント提案書と矛盾する |
| 売上・粗利などの財務情報 | 経営情報 |
| クライアント未許諾の社名 | 守秘義務 |
| 技術的な内部実装の詳細 | 競合への情報開示 |

### 4.3. 今すぐ発信できるネタ

| # | ネタ | チャネル | 状態 |
|:---|:---|:---|:---|
| 1 | **Rakuco導入事業所100社突破** | note + プレスリリース + X | ✅ すでに事実。鮮度あり |
| 2 | **株式会社への組織変更** | note + X | ✅ すでに事実 |
| 3 | **コーポレートサイトWorksページ追加** | X | ✅ 実装済み |

## 5. 運用フロー

### 5.1. 日常（随時）

```
ネタが発生（リリース・マイルストーン・契約etc.）
  ↓
X or noteで発信（10分以内）
  ↓
ネタリストに記録（スプレッドシート or Notion）
```

### 5.2. 四半期（3ヶ月に1回）

```
ネタリストから3〜5件ピックアップ
  ↓
LLMでニュースレター下書き生成
  ↓
池田がレビュー（15分）
  ↓
配信ツールで送信
```

### 5.3. 自動化の方針

| フェーズ | 内容 | 池田の工数 |
|:---|:---|:---|
| **Phase 1（今）** | ネタ発生時にX投稿 + スプレッドシートに記録 | 月30分 |
| **Phase 2** | LLMでニュースレター下書き生成。池田は確認・送信のみ | 四半期15分 |
| **Phase 3** | 開発副産物から自動ネタ抽出 → X投稿案 + ニュースレター素材を自動生成 | 承認のみ |

## 7. テックブログとの関係

テックブログ（blog.soct.jp）は**素材蓄積フェーズ**として保留中。以下の条件が揃った時点で再検討する。

| トリガー | 時期（目安） |
|:---|:---|
| Tetra LMSのリリース + 初期テナント獲得 | Q3以降 |
| テレアポ分析の定量成果が出た | 計測開始から3ヶ月後 |
| 中川様案件の完了 + 公開許可 | Q4以降 |

テックブログが立ち上がった場合、ニュースレターから記事へリンクする形で統合する。
