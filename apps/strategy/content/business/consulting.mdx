## 1. 事業概要

**障害福祉事業所（就労継続支援A型・B型）の経営者に対し、月額¥50,000の顧問契約で「増収支援」と「減算リスク回避」を中心とした伴走型コンサルティングを提供するサービス。**

### サービスの全体像

| 項目 | 内容 |
|:---|:---|
| **対象** | 障害福祉事業所（就労継続支援A型・B型）の経営者・管理者 |
| **提供価値** | ①未取得加算の発掘による増収 ②減算リスクの早期発見・回避 ③経営基盤の強化（Notion導入・集客・人材育成等） |
| **月額** | ¥50,000/社（固定） |
| **提供方法** | 月1回の定例オンラインMTG（1時間）+ Notionによるタスク管理・進捗共有 |
| **成果イメージ** | 加算取得による年¥500K〜¥2Mの増収 / BCP・情報公表等の減算回避 / 業務の可視化 |
| **運営** | 株式会社はなえみ（北澤：専門コンサル）× 株式会社SOCT（山田：PM + AI基盤開発） |

```
クライアントが得る体験:

  月1回の定例MTG
    ├── 「取れるのに取っていない加算」の発掘 → 増収
    ├── 「知らないうちに減算されている」リスクの回避 → 損失防止
    ├── 経営課題の壁打ち（集客・人材・DX等） → 経営力向上
    └── Notionで「今何をすべきか」が常に見える → 業務改善

  → 顧問料¥600K/年 に対して、加算増収だけで年¥500K〜¥2Mのリターン
```

### 商談中（プロスペクト）にやること — 2回商談モデル

> **目的: 契約獲得。** 初回でヒアリング → レポート作成 → 2回目でレポートを見せて提案。

**初回商談（ヒアリング）**

| やること | 内容 |
|:---|:---|
| **ヒアリング** | [ヒアリングシート](../guides/hearing-sheet)に沿ってA型/B型別にデータ収集 |
| **次回日程の確定** | 「分析レポートのご説明」として1〜2週間後に設定 |

**2回目商談（レポート提示・提案）**

| やること | 内容 |
|:---|:---|
| **運営分析レポートの提示** | 事業所の現状診断を画面共有で見せる。**データは渡さない** |
| **改善の方向性の提案** | 「こういう加算が取れそう」「この減算リスクに対応が必要」等の口頭提案 |
| **契約メリットの説明** | 月額¥50Kで何が得られるかを具体的に説明 |

**やらないこと**

| やらないこと | 理由 |
|:---|:---|
| 具体的なアクションプランの策定 | 契約後のサービス |
| 書類のテンプレート提供 | 契約後のサービス |
| 運営分析レポートのデータ提供 | 契約後にNotionで共有。商談時は見せるだけ |

### 契約後（顧問先）にやること

> **目的: 継続支援 + 解約防止。** 月1回の定例MTGとNotionで伴走する。

| やること | 頻度 | 内容 |
|:---|:---|:---|
| **Notionクライアントページの運用** | 常時 | TODO・議事録・月次報告を一元化。クライアントはURLで閲覧 |
| **定例MTG** | 月1回（1時間） | TODO消化確認 → 加算・減算の進捗 → 経営課題の壁打ち |
| **加算取得支援** | 随時 | 未取得加算の要件確認・テンプレート提供・申請方法の助言 |
| **減算リスク回避** | 随時 | BCP・情報公表・虐待防止等の対応状況確認 |
| **月次報告** | 月1回 | 今月の成果・来月のアクション・改善効果の可視化（Notion内） |
| **一般課題支援** | 随時 | Notion社内導入・集客・人材育成等（§5.7 参照。解約防止策） |

> **Notion運用方針:**
> - **ワークスペースはこちら（はなえみ/SOCT）側で一元管理する。** クライアントにNotionアカウント作成を求めない。共有URLでの閲覧のみ。
> - 100社スケール時のテンプレート統一・MCP/AI連携を見据え、全クライアントを1つのWSで管理する。
> - **クライアント社内でのNotion導入支援（社内DX）は別サービス。** §5.7 の一般課題支援（解約防止・LTV最大化）として位置づける。

### 1.1. 本ドキュメントの目的

> 上記サービスの**内部運営設計書**。以下を定義する。

1. 北澤（メインコンサル）と山田（PM/サポート）の役割分担を明確にする
2. 事前調査を徹底し、初回商談の成約率（クロージング率）を最大化させる
3. **SOCTのAI/自動化技術をバックエンドに据え、省力化と属人化排除により100社を支援可能な体制を構築する**
4. 全行程をNotionで型化し、支援の質を落とさず対応件数を増やす

### 1.2. 戦略的位置づけ — AI-Augmented Consulting

> [はなえみダイレクト事業計画](./hanaemi-direct) の **収益軸③（コンサル・研修）** の実行プロセス。
> [クロスセルアイデア](../plans/cross-sell-ideas) の「SOCT × 株式会社はなえみ」連携モデルの具体化。

**設計思想: 「表はコンサル、裏はAIオペレーション」**

> ⚠️ **戦略方針（2026-04-21 確定）:** 単価¥50K/社を固定し、**省力化と属人化排除により100社を回す**ことを目標とする。§4のAI自動化は「将来構想」ではなく、100社到達の**必須条件**。北澤の専門知識を型化・AI化し、複数人で回せる体制を構築する。

従来のコンサルティングは「専門家の経験と時間を売る」モデルのため、個人の稼働がそのまま売上の天井になる。SOCTはここに**AI/自動化技術をバックエンドとして組み込む**ことで、従来型コンサルの**10倍以上の案件を同じ人数で回す**。

```
クライアントから見える世界（フロントエンド）:
  ┌─────────────────────────────────────────────┐
  │  北澤の専門知識による対面コンサル              │
  │  山田のPM管理・月次レポート                   │
  │  → 「手厚い伴走型の支援」に見える              │
  └─────────────────────────────────────────────┘
                      ▲
                      │ 価値提供
                      │
クライアントに見えない世界（バックエンド = SOCTの技術力）:
  ┌─────────────────────────────────────────────┐
  │  🤖 ワムネットデータ自動収集・分析              │
  │  🤖 加算ギャップの自動検出・改善効果の試算                  │
  │  🤖 議事録→FAQ/ナレッジの自動抽出              │
  │  🤖 月次報告書の自動生成                       │
  │  🤖 監査チェックリストの自動照合                │
  │  🤖 報酬改定差分の自動アラート                  │
  │  → 北澤の判断時間を「分析」から「意思決定」に集中 │
  └─────────────────────────────────────────────┘
```

> **これがSOCTでなければできない理由:** 障害福祉SaaS（Rakuco）の開発で蓄積した**制度の構造化知識** + **AI/自動化の開発力**（[フォーム営業設計書](../plans/form-sales-system)と同じ思想）の組み合わせ。行政書士事務所にもコンサル会社にもない能力。

```
SOCTの事業ポートフォリオにおける位置:

  SOCT（テック × スケーラブル）
    ├── Rakuco           ── SaaS ──────────┐
    ├── はなえみラーニング  ── eラーニング     │ 制度知識
    ├── Tetra LMS         ── プラットフォーム  │ の共有
    └── はなえみダイレクト  ── マッチングPF    │

                           │
  株式会社はなえみ（人的サービス × 深い支援）
    └── 顧問サービス ── 制度対応・経営相談 ← 北澤が実施
```

### 1.4. 収益構造（はなえみダイレクト §8.3 準拠）

> ⚠️ 収益配分モデルは方針ほぼ確定。正式合意後に確定。

**単価:** 月額 **¥50,000/社**

#### 現在の売上実態（2026-04-21 時点）

| 区分 | 社数 | 月間売上 | 備考 |
|:---|:---|---:|:---|
| 既存クライアント（定例4社） | 4社 | ¥200K | グリーン、オリィーブ、トモイキ、はたむーぶ |
| 新規パイプライン（商談段階） | 2社 | — | アンスーリール、若葉/大田様 |
| **コンサル売上計** | **4社（+2社見込み）** | **¥200K（最大¥300K）** | — |

> ⚠️ **要確認:** 既存4社がSOCT参画前からの顧客か、参画後の獲得かで「増分」の算出が変わる。

#### 収益配分モデル

| はなえみ月間売上 | SOCTの取り分 | 社数 | フェーズ |
|:---|---:|:---|:---|
| ¥400K（ベースライン） | **¥0** | 既存4社程度 | 現状 |
| ¥700K | **¥300K** | +6社（計10社） | **Phase 1 ← SOCT損益分岐** |
| ¥1,500K | **¥750K** | 約30社 | Phase 2 |
| ¥3,000K | **¥1,500K** | 約60社 | Phase 3 |
| **¥5,000K** | **¥2,500K** | **100社** | **Phase 4 = 最終目標** |

> **算出根拠:** ¥800Kまでは増分100%→SOCT、¥800K超は50/50。¥5,000Kの場合: ¥400K +（¥5,000K - ¥800K）× 50% = ¥2,500K/月（年¥30M）。

> **核心:** 山田のPF営業兼務によるコンサル案件の新規獲得分がSOCTの収益。はなえみ売上が¥800K/月に達するまでは増分100%をSOCT、到達後は50/50。

> ⚠️ **注意:** 現行の配分モデルは¥800K程度の規模を想定して設計された。Phase 2（30社/¥1,500K）以降では、ジュニアコンサルの人件費負担やAI開発投資の回収を含めた**配分モデルの再設計**が必要になる。Phase 1到達時に北澤と正式に再協議する（§9 #1）。

#### 100社到達ロードマップ

| Phase | 社数 | 体制 | 1社あたり工数 | 必要な自動化 | 目標時期 |
|:---|:---|:---|:---|:---|:---|
| **Phase 0（現在）** | 6社 | 北澤（1人）+ 山田PM | 8〜10h/社 | なし（手動） | 現在 |
| **Phase 1** | 10〜15社 | 北澤 + 山田PM + AI | 4〜5h/社 | 運営分析レポート自動生成・議事録AI要約 | Q3 2026 |
| **Phase 2** | 30社 | 北澤 + ジュニアコンサル1名 + AI | 2〜3h/社 | 加算チェック自動化・月次報告自動生成 | Q1 2027 |
| **Phase 3** | 60社 | 北澤(監修) + コンサル2名 + AI | 1.5〜2h/社 | 全フェーズ自動化・北澤ナレッジDB化 | Q3 2027 |
| **Phase 4** | **100社** | 北澤(監修) + コンサル3〜4名 + AI | **1h/社** | 完全自動化。人は「MTG + 判断」のみ | 2028 |

> **スケールの鍵:** 北澤の専門知識を「型」にしてAIに載せること。100社時点で北澤は「監修・難案件対応」に専念し、定常支援はAI + ジュニアコンサルが回す。

#### 単価の戦略的意義

| 観点 | 分析 |
|:---|:---|
| **¥50K固定の理由** | 行政書士・社労士の顧問料相場帯（¥30K〜¥80K）の中間。導入障壁が低く、量で勝負する戦略に最適 |
| **クライアント視点のROI** | §5の加算取得支援で年¥500K〜¥2M増収が可能。顧問料¥600K/年に対し十分なリターン |
| **競合優位性** | AI自動化による原価圧縮で、¥50Kでも¥80K以上のサービス品質を提供。競合はコスト的に追随不能 |

### フェーズ1：初回商談 = ヒアリング（北澤主導 / 山田同席）

> **目的: データ収集。** ワムネットで取れない情報をヒアリングで補完する。この場では答え（分析結果・数字の約束）は出さない。

| # | プロセス | 担当 | 補足 |
|:---|:---|:---|:---|
| 1 | **アイスブレイク + 目的説明** | 北澤 | 「御社の状況を伺い、分析レポートを作成してお持ちします」 |
| 2 | **ヒアリング** | 北澤+山田 | [ヒアリングシート](../guides/hearing-sheet)に沿ってA型/B型別に情報収集 |
| 3 | **次回日程の確定** | 山田 | 「分析レポートのご説明」として1〜2週間後に設定 |
| 4 | **記録** | 山田 | 議事録をリアルタイムで作成 |

> **ヒアリングの原則:** [テレアポガイド §B](../guides/teleapo-talk-guide) の「ヒアリングが先、提案が後」を厳守。初回商談では売り込まない。

---

### フェーズ2：運営分析レポート作成 + 内部打ち合わせ

**レポート作成（山田 / AI補助）**

| # | タスク | 詳細 |
|:---|:---|:---|
| 1 | **データ統合** | ワムネット公開データ + ヒアリング結果を統合 |
| 2 | **運営分析レポート作成** | AIに投げ、攻め・守り・経営基盤の3面で診断をまとめる |
| 3 | **レポートを北澤に共有** | Notionまたはドキュメントで事前送付 |

**内部打ち合わせ（山田 × 北澤）🔥 最重要**

| 項目 | 内容 |
|:---|:---|
| **時間** | 2回目商談の前日までの15〜30分 |
| **決定事項** | レポートの各項目を確認し、**談論の優先順位と濃淡**を決める |

> **内部打ち合わせで決めること:**

## 5. 加算・減算の網羅的チェック体系

### 5.1. 攻めの加算 — 重点確認項目

> 処遇改善加算は全クライアントで対応中のため省略。以下は**コンサルが介在する価値の高い加算のみ掲載**。
> 食事提供体制加算・送迎加算等の自明な加算は省略。**網羅的な加算リストは北澤ナレッジDB化（#14）+ AIモジュールで対応予定。**

| # | 加算名 | 対象 | 概要 | 想定増収（20人規模） | コンサル付加価値 |
|:---|:---|:---|:---|---:|:---|
| 1 | **福祉専門職員配置等加算** | A/B型 | 社会福祉士等の資格保有者の配置割合 | 配置依存 | 有資格者がいるのに未申請のケースが多い。ワムネットで事前検出可能 |

### 5.2. 守りの減算 — 3大リスク

> ⚠️ 以下の3大減算は、議事録で**一切議論されていない**。経過措置終了済みのため、未対応のクライアントがいれば遡及適用で報酬返還リスクあり。

| # | 減算名 | 影響 | 概要 | 対応 |
|:---|:---|---:|:---|:---|
| 1 | **BCP未策定減算** | 所定単位の1〜3% | 感染症 + 自然災害の両方のBCPが必要。経過措置は令和7年3月終了済み | BCPテンプレ提供→カスタマイズ→北澤レビュー |
| 2 | **情報公表未報告減算** | 基本報酬減算 | ワムネット等への情報公表制度に基づく報告が未実施の場合 | 公表状況を確認→未報告項目の洗い出し→期限内報告支援 |
| 3 | **虐待防止措置未実施減算** | 所定単位の1% | ①虐待防止委員会の設置・年1回以上開催 ②職員研修の定期実施 ③担当者配置 の3点セット | チェックリストで3点の実施状況確認→未実施項目の対応支援 |

### 5.7. 中小企業の一般課題への支援

> **2026-04-21 新設。** 議事録を精査すると、加算・減算といった業界固有の論点以外に、**中小企業なら共通の経営課題**が複数見えてくる。これらはSOCTのテック力が直接活きる領域であり、「制度対応専門家」を超えたコンサルの差別化要因になる。

#### 議事録から見えるシグナル

| 議事録の記述 | クライアント | 背景にある一般課題 |

#### SOCTが提供可能な一般課題支援メニュー

| # | 支援領域 | 内容 | SOCTの強み | 課金機会 |

> **戦略的意義:** これらの一般課題支援は、単なる「追加サービス」ではなく、**コンサルの解約防止とLTV最大化の基盤**になる。
>
> ```
> 現在: 加算・減算の制度対応のみ
>   → 「制度のことがわかったら卸業」のリスク
>
> 目指す姿: 制度対応 + 経営基盤の強化
>   → Notionで業務が回り、集客が安定し、人が育つ
>   → 「この人たちがいないと困る」状態 = 高いLTV
> ```
>
> **スケールへの貢献:**
> - これらの一般課題支援は「制度のことがわかったら卸業」を防ぐ**解約防止策**として機能
> - Notion導入・集客支援は**型化・テンプレート化が容易**なため、100社スケール時も少ない工数で提供可能
> - 「¥50Kでここまでやってくれる」という**圧倒的なコスパ**が口コミを生み、営業コストを下げる

## 8. コンプライアンス・法的リスク

> **2026-04-21 新設。** 障害福祉コンサルティングは、士業の独占業務に抵触するリスクがある。100社スケール時にジュニアコンサルが増えるため、「やっていいこと／やってはいけないこと」の境界を明確にしておく。

### 8.1. 士業法との境界整理

| リスク | 関連法 | 抵触する行為 | 本サービスでの対応方針 |
|:---|:---|:---|:---|
| **非弁行為** | 弁護士法72条 | 法律事件に関する代理・仲裁・和解等を業として行うこと | 行政との紛争・係争の「代理」は行わない。助言・同席までに留める |
| **社労士法違反** | 社会保険労務士法27条 | 労働社会保険の手続き書類の作成・提出代行を業として行うこと | 処遇改善加算の計画書・就業規則等の**作成代行は行わない**。作成方法の助言・テンプレート提供・レビューに留める |
| **行政書士法違反** | 行政書士法19条 | 官公署に提出する書類の作成を業として行うこと | 加算届出書類・BCP等の**作成代行は行わない**。記載例の提供・チェックリストでの確認に留める |
| **税理士法違反** | 税理士法52条 | 税務書類の作成・税務相談を業として行うこと | 税務に関する具体的な助言は行わない。運営分析レポート内の試算は「報酬単位の見積もり」であり税務相談ではない |

### 8.2. 「助言OK／代行NG」の原則

```
✅ やっていいこと（助言・情報提供）:
  ├── 「この加算を取得すれば年¥○○万の増収が見込めます」 → 情報提供
  ├── 「BCPのテンプレートはこちらです。こう記載してください」 → テンプレ提供
  ├── 「処遇改善計画書の記載例をお見せします」 → 記載例の提示
  ├── 「行政に確認した方がいいポイントはここです」 → 助言
  └── 「〇〇加算の要件はこうです」 → 制度の解説

❌ やってはいけないこと（代行・代理）:
  ├── クライアントの計画書・届出書類を書いて提出する → 行政書士法違反
  ├── 就業規則を作成して労基署に届出する → 社労士法違反
  ├── 行政との交渉・紛争を代理する → 弁護士法違反
  └── 税務申告や具体的な節税アドバイスをする → 税理士法違反
```

### 8.3. リスク軽減策

| # | 対策 | 詳細 |
|:---|:---|:---|
| 1 | **顧問契約書に明記** | サービス範囲を「助言・情報提供・テンプレート提供」に限定し、書類作成代行・行政代理は含まないことを明記 |
| 2 | **士業との連携体制** | 書類作成が必要な場合は提携の行政書士・社労士を紹介する。スケール時は提携士業事務所との包括契約を検討 |
| 3 | **ジュニアコンサル研修** | Phase 2以降の採用時研修に「士業法の基礎」を必須科目として組み込む（§2.3 参照） |
| 4 | **定期レビュー** | 半年に1回、提供サービスの内容が士業の独占業務に抵触していないかを確認 |

> ⚠️ **100社スケール時の重要性:** 北澤は業界経験から境界を感覚的に理解しているが、ジュニアコンサルはそうではない。「テンプレートを渡して書き方を教える」と「代わりに書いて提出する」の一線を超えると法的リスクが発生する。ナレッジDB（§9 #14）にガードレールを組み込む必要がある。
