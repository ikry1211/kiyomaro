# Rakuco M&A戦略

> **ステータス:** 検討中（M&Aセンターと協議開始）
> **最終更新:** 2026-03-22
> **方針:** Rakuco単体売却 / eラーニングはSOCT保有 / 2027年7月以降クロージング

---

## 1. 売却方針

| 項目 | 方針 |
|:---|:---|
| 売却対象 | **Rakuco事業のみ**（eラーニング・Tetra LMS・はなえみダイレクトは含めない） |
| 売却時期 | **2027年7月以降**（2027年度報酬改定対応後） |
| 目標売却額 | **¥1.5億以上**（最低手数料¥2,000万を考慮し、¥1.2億が下限） |

### eラーニングを含めない理由

1. **ARRマルチプルベースなら、Rakuco単体のARR ¥3,830万 × 4倍 = ¥1.53億で¥1.2億をクリア**
2. eラーニングは立上段階。含めるとDDが複雑化し、不確実性のディスカウントがかかる
3. eラーニングはTetra LMS基盤で独立運営可能。売却後もSOCTの収益源として残せる
4. Rakuco顧客との関係（eラーニングのクロスセルチャネル）もSOCT側に維持できる

### 売却タイミングの合理性（買い手視点）

> ⚠️ **「2027年報酬改定対応後」に売る最大のメリット：買い手にとって向こう3年間改定がない。**

| 比較 | 2026年秋に売った場合 | **2027年7月に売った場合** |
|:---|:---|:---|
| 買い手の直後のリスク | 2027年度改定対応が必要 | **3年間改定なし（〜2030年）** |
| 開発引継ぎ | 改定対応を含む → 高リスク | 改定完了済み → 安定運用のみ |
| マルチプルへの影響 | ディスカウント要因 | **プレミアム要因** |

売却時点には直近6ヶ月分のトライアル顧客が滞留しており、「ほぼ確定的な将来収益」として評価交渉の材料になる。

## 4. マルチプルを引き上げるための16ヶ月アクション

| ドライバー | 現状 → 目標 | アクション |
|:---|:---|:---|
| **NRR（売上維持率）** | 100%未満 → **110%+** | 値上げ¥18K→¥21Kで117%。最も倍率に効く |
| **チャーン率** | 0.96% → **0.5%以下** | QA強化で12月の6社一括解約パターンを再発防止 |
| **SaaS粗利率** | 86.4% → **91%+** | インフラバグ修正でCOGS低減 |
| **規制安定性** | 改定控え → **3年安定** | 2026臨時+2027通常の両方を完了して売却 |
| **営業体制** | CPA改善中 → **実績の可視化** | 直近3月CPA ¥71K・LTV/CAC 13.6倍。再現性ある営業エンジンとしてアピール |

### 四半期別タイムライン

```
Q2 (2026/4〜6)  報酬改定対応、QA強化、インフラバグ修正開始
Q3 (2026/7〜9)  粗利91%+達成、値上げ準備、解約予兆モニタリング
Q4 (2026/10〜12) 値上げ実施（¥18K→¥21K）、技術ドキュメント・契約書類整備
Q5 (2027/1〜3)  DD用財務資料整備、運用マニュアル整備

2027/7〜        契約締結・クロージング
```
